Student Projects

Hands-on exercises to take you from 1D to 3D — choose your own path or work with an assigned project

These projects are for after the summer school. Take them home and work through them at your own pace. Questions in the meantime? Email Sebastian (sebastian.schmidt@lasp.colorado.edu).
Before you start: Make sure EaR³T is installed and the verification script passes. Each project has a Jupyter notebook starter — clone the course repository and open the notebook in your conda environment. You should be able to complete the core tasks in 1.5 hours; the "going further" sections are for those who want to dig deeper.

Overview

All projects follow the same arc: start with real satellite data, set up a 3D-RT simulation, compare the simulation to an independent observation, and interpret the difference. Each project targets a different aspect of 3D radiative effects — from surface irradiance prediction to radiance closure to sub-pixel cloud structure.

Prerequisites: Python (intermediate), Jupyter notebooks, basic familiarity with atmospheric science and radiative transfer concepts (covered in the intro lecture). No prior experience with 3D RT needed.

A
Arctic Surface Irradiance from Polar Satellite Data
Pituffik, NW Greenland · MODIS/VIIRS → surface insolation
MODIS · VIIRS Surface radiometer EaR³T irradiance ⭐⭐ Intermediate

Use a polar-orbiting satellite overpass over Pituffik (Thule Air Base, NW Greenland) as the starting point. EaR³T will download the MODIS or VIIRS cloud retrieval, set up 3D-RT, and simulate downwelling shortwave irradiance at the surface. You then compare the prediction with actual surface radiometer observations (e.g., from the ARCSIX campaign or BSRN network).

You will learn: How to drive EaR³T with polar satellite data; how cloud optical thickness and cloud fraction affect surface insolation; what the difference between 1D and 3D simulations looks like for Arctic cloud scenes (often extended stratiform cloud decks — a relatively 1D-friendly case).

Going further: Compare multiple overpasses with different cloud conditions. Identify which scenes show the largest 1D–3D difference and explain why.

B
Aircraft Irradiance Along a Flight Track
Geostationary or polar imagery → airborne SP-NS comparison
AHI · MODIS Airborne SP-NS EaR³T flight track ⭐⭐ Intermediate

Following the approach of Chen et al. (2023, App. 3), use geostationary or polar-orbiting cloud retrievals to simulate downwelling irradiance along an aircraft flight track, then compare with direct airborne SP-NS measurements. The CAMP²Ex dataset (Philippines, 2019) or ARCSIX data can be used as a test bed.

You will learn: How campaign-scale irradiance closure works; how the coarse resolution of geostationary imagers introduces biases; the statistical distribution of 1D–3D irradiance differences across different cloud regimes.

Going further: Stratify the results by cloud fraction and optical thickness. Show which flight segments have the largest biases and map them back to the satellite imagery.

C
Radiance Closure with WorldView Cloud Scenes
NASA WorldView → IPA retrieval → 3D-RT → closure + irradiance bias
NASA WorldView MODIS L1B/L2 EaR³T radiance sim ⭐⭐⭐ Advanced

This is the core concept of Chen et al. (2023). Pick an inhomogeneous cloud scene from NASA WorldView (interactive imagery browser). Use EaR³T to:

  1. Download the MODIS cloud retrieval (IPA-derived COT and CER fields)
  2. Run a 3D-RT forward simulation to predict what radiance the satellite should have seen
  3. Compare the simulated radiance with the actual MODIS L1B measurement — this is the closure test
  4. Estimate the surface irradiance bias implied by the COT bias you detected

You will learn: The full radiance closure workflow; what makes a cloud scene 3D-RT-biased vs. IPA-consistent; how radiance bias propagates into irradiance and cloud radiative effect errors.

Advanced variant — CNN retrieval: Apply the EaR³T-included CNN (Nataraja et al. 2022) to the scene and compare its radiance closure metrics against the IPA. Does the CNN achieve better self-consistency?

D
Sub-Pixel Cloud Inhomogeneity
What does the satellite retrieve when a cloud under-fills a pixel?
Configurable LES scenes EaR³T 3D sim ⭐⭐⭐ Advanced

Simulate an imager pixel that is only partially filled by cloud (a "broken" cloud field at the sub-pixel scale). Use EaR³T to generate the pixel-integrated radiance, then ask: what does a standard 1D retrieval return? It will find a cloud optical thickness somewhere between zero and the true value — but it has no way to separately retrieve cloud fraction and optical thickness.

Then explore whether spectral information can help break this degeneracy. Multi-channel reflectance from an imaging spectrometer carries a signal that, in principle, allows partial separation of cloud fraction and optical thickness. Test this hypothesis using EaR³T radiance simulations across a range of scene configurations.

You will learn: The sub-pixel cloud fraction problem; the limits of passive imagery retrievals; spectral information content analysis.

Scene progression: Start with a fully overcast stratus layer (1D limit), then move to broken stratocumulus, then to a isolated cumulus field — and track how the retrieval bias evolves.

E
3D Effects on Passive Cloud Vertical Structure Retrievals
Research track · spectral reflectance + LES + information content
LES cloud scenes EaR³T 1D + 3D 🔬 Research-grade

New methods (Buggee et al., CU Boulder) retrieve the vertical structure of clouds from passive shortwave reflectance spectra. This project asks: does 3D radiative transfer introduce sufficient spectral perturbations to reduce the information content of these retrievals?

  1. Take 3D LES cloud scenes (stratus → stratocumulus → cumulus)
  2. Use 1D-RT (libRadtran) to generate synthetic spectral reflectance as a "traditional" satellite would observe
  3. Use 3D-RT (MCARaTS via EaR³T) to generate the true spectral radiance field
  4. Quantify the spectral difference and assess whether it degrades the vertical structure retrieval

You will learn: Spectral information content analysis; how 3D effects perturb reflectance spectra; the connection between cloud morphology and retrieval uncertainty.

This project is also a personal research interest of Sebastian's — results may feed directly into a future publication. Don't be shy about going deep.

Tips for the 1.5-Hour Session

  • Pick one project and focus on getting the core result — don't try to do everything.
  • If the data download is slow, use pre-cached data (will be provided).
  • The "going further" sections are for bonus time — they're open-ended on purpose.
  • Work with a partner if you like — discussion is encouraged.
  • Sebastian will circulate — ask questions early rather than getting stuck.
