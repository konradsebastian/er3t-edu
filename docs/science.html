Science Introduction & Context

From 1D radiative transfer to a 3D understanding of clouds, photons, and remote sensing

Why Does Dimensionality Matter?

Radiative transfer — the physics of how radiation propagates through the atmosphere — is at the heart of remote sensing, climate science, and weather forecasting. Nearly every operational satellite retrieval algorithm uses a one-dimensional (1D) approximation: it treats each pixel as if the atmosphere above it were an infinite, horizontally uniform layer.

This is the Independent Pixel Approximation (IPA), and it has served the community well for decades. But the real atmosphere is not 1D. Clouds have edges, towers, shadows, and gaps. Photons scatter sideways — out of one pixel and into another. These horizontal photon transport effects create systematic biases in retrieved cloud properties and surface irradiance that the 1D framework simply cannot capture.

The core problem: When we retrieve cloud optical thickness using 1D radiative transfer, the algorithm finds the cloud thickness that makes a 1D model match the measured radiance. But if the photons that hit the sensor came partly from neighboring pixels, the 1D model will return the wrong answer — and it will do so systematically, in ways that depend on cloud morphology.

The 1D Baseline: libRadtran

The most widely used 1D radiative transfer package in the atmospheric science community is libRadtran. It provides a flexible, user-friendly interface to a range of 1D solvers (DISORT, RODENT, TWOSTR, and others) and handles atmospheric gas absorption, aerosol optical properties, surface reflectance, and more.

EaR³T is inspired by libRadtran and uses it as one of its solvers — specifically to achieve a consistent 1D benchmark against which 3D results can be compared. This means the difference between a libRadtran run and an MCARaTS 3D run through EaR³T gives you a clean, well-defined measure of the 3D effect.

Student version: The teaching flavor of EaR³T is intentionally lightweight. It uses MCARaTS as the sole solver and does not require a libRadtran installation. All seven examples run with MCARaTS only, using EaR³T's built-in gas absorption parameterization (REPTRAN, included in the data package). LibRadtran is only needed if you move on to research use.

Stepping Into 3D: EaR³T and MCARaTS

EaR³T wraps a set of 3D radiative transfer solvers, most prominently MCARaTS (Monte Carlo Atmospheric Radiative Transfer Simulator by H. Iwabuchi). Monte Carlo methods trace individual photon paths through a fully 3D atmospheric grid, capturing all horizontal transport effects. The cost is computation time — but for modern research and education, it is tractable on a laptop or a small cluster.

EaR³T automates the tedious parts: downloading satellite imagery, converting cloud retrievals into 3D optical property grids, setting up the solver input, running it, and collecting output radiances and irradiances. This lets students and researchers focus on the physics rather than the plumbing.

The Radiance Closure Concept

At the core of EaR³T's scientific approach is radiance closure (or radiance self-consistency). The idea is straightforward:

  1. Take a satellite measurement of radiance Robs.
  2. Run a retrieval algorithm (using 1D-RT) to get cloud properties (optical thickness, effective radius, etc.).
  3. Use those retrieved properties as input to a 3D-RT forward model and simulate what radiance the satellite should have seen: Rsim,3D.
  4. Compare Robs and Rsim,3D. If they match, the retrieval is 3D-consistent. If not, the gap is a direct measure of the 3D-RT retrieval bias.
Key insight: Traditional retrievals guarantee consistency with 1D-RT (by construction). EaR³T tests whether they are consistent with 3D-RT (i.e., with nature). Inconsistency = bias.

Applications as documented in the literature

1. OCO-2 Radiance Closure

EaR³T generates synthetic radiances at OCO-2 wavelengths from MODIS cloud retrievals and compares them to actual OCO-2 measurements. Over inhomogeneous cloud scenes (e.g., SW Colorado), significant inconsistency reveals 3D-RT biases in the input cloud products. (Chen et al. 2023)

2. MODIS Radiance Self-Consistency

MODIS L2 cloud products are used as input to 3D-RT to simulate MODIS L1B radiances. IPA cloud optical thickness retrievals are found to be biased low due to horizontal photon transport — a bias that comes to light only when 3D-RT is used for the forward simulation. (Chen et al. 2023)

3. Campaign-Scale Irradiance Closure

EaR³T processes geostationary (AHI) cloud retrievals for the full CAMP²Ex aircraft campaign (NASA 2019, Philippines) and simulates irradiance along every flight leg. Comparison with SP-NS airborne measurements reveals a systematic 10% low bias in satellite cloud transmittance, attributable to a combination of coarse GEO resolution and 3D-RT effects. (Chen et al. 2023)

3a. Surface Solar Irradiance in Shallow Cumulus Cloud Fields

A series of studies uses EaR³T together with large-eddy simulation (LES) to understand how shallow cumulus cloud fields control surface solar irradiance. The first paper establishes that simple cloud-field statistics (cloud fraction, optical depth) are insufficient predictors of surface irradiance — horizontal photon transport introduces systematic deviations from the 1D expectation. (Gristey et al. 2020, JAS) A follow-up study quantifies how these deviations scale with cloud morphology properties. (Gristey et al. 2020, GRL) A third paper extends the framework to aerosol effects: when absorbing or scattering aerosol is embedded in shallow cumulus, the 3D irradiance response diverges substantially from 1D predictions, with implications for aerosol radiative forcing estimates. (Gristey et al. 2022, JGR)

4. CNN Cloud Retrievals

EaR³T generates large synthetic datasets (LES cloud fields → 3D-RT radiances) to train a U-Net convolutional neural network that retrieves cloud optical thickness while accounting for spatial context. The CNN substantially reduces IPA bias across all cloud morphologies. (Nataraja et al. 2022)

5. OCO-2 CO₂ Bias Correction

Nearby clouds contaminate OCO-2 XCO₂ retrievals via 3D scattering into cloud-free footprints. EaR³T-OCO computes 3D-RT spectral perturbations and applies them as a pre-correction to OCO-2 spectra before retrieval — the first physics-based 3D correction for a trace-gas satellite. Residual biases are reduced from >2 ppm to <0.7 ppm (regional average). (Yu-Wen Chen et al. 2024)

The Big Picture

The overarching theme of EaR³T research is the transition from an oversimplified 1D view of the world to a fully 3D-aware one. The drivers are:

  • New, fast 3D solvers (MCARaTS, SHDOM) that make 3D-RT computationally feasible for real scenes
  • High-resolution satellite imagery (MODIS, VIIRS, AHI, OCO-2) providing the 3D cloud input fields
  • Machine learning as a way to encode 3D-RT knowledge without running full simulations in real time
  • New multi-sensor mission architectures (EarthCARE, AOS) that are specifically designed for 3D radiance closure
